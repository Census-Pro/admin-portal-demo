# NDI QR Code Generation Troubleshooting

## Issue: Stuck on "Generating QR Code..."

The QR code is **generated by the NDI service**, not by our application. The flow is:

```
Admin Portal ŌåÆ Auth Service ŌåÆ NDI Service (Bhutan NDI) ŌåÆ Returns QR Code URL
```

## Common Causes

### 1. Auth Service Not Running

**Symptom:** Spinner keeps running, no console errors initially

**Check:**

```bash
# Is auth service running?
curl http://localhost:5001/health

# Expected: {"status":"ok",...}
```

**Fix:**

```bash
cd auth_service
pnpm dev
```

---

### 2. NDI Environment Variables Not Configured

**Symptom:** Console shows `NDI_VERIFIER_URL not configured` or similar

**Check auth service `.env`:**

```bash
cd auth_service
cat .env | grep NDI
```

**Required Variables:**

```bash
NDI_AUTH_URL=https://staging.bhutanndi.com/authentication/v1/authenticate
NDI_CLIENT_ID=your_client_id_here
NDI_CLIENT_SECRET=your_client_secret_here
NDI_VERIFIER_URL=https://staging.bhutanndi.com
NDI_SCHEMA_NAME=Foundational ID
NDI_NATS_URL=nats://staging.bhutanndi.com:4222
NDI_NATS_SEED=your_nkey_seed_here
```

**Fix:** Add/update the NDI variables in `auth_service/.env`

---

### 3. NDI Service Unreachable

**Symptom:** Console shows network timeout or connection refused

**Check:**

```bash
# Test NDI authentication endpoint
curl -X POST https://staging.bhutanndi.com/authentication/v1/authenticate \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "your_client_id",
    "client_secret": "your_client_secret",
    "grant_type": "client_credentials"
  }'

# Expected: {"access_token":"...","expires_in":3600,...}
```

**Fix:**

- Check internet connection
- Verify NDI service URLs
- Contact NDI support if service is down

---

### 4. Invalid NDI Credentials

**Symptom:** Console shows `Authentication failed: 401` or `403`

**Check auth service logs:**

```bash
cd auth_service
pnpm dev

# Look for:
# ŌØī Failed to authenticate with NDI
# Authentication failed: 401 - Unauthorized
```

**Fix:**

- Verify `NDI_CLIENT_ID` and `NDI_CLIENT_SECRET` are correct
- Contact NDI support to verify credentials
- Check if credentials expired

## Understanding QR Code Generation

**The QR code is NOT generated by our code.** Here's what happens:

1. **Our code** sends a proof request to NDI service
2. **NDI service** creates the proof request and generates a QR code
3. **NDI service** returns a URL where the QR code image is hosted
4. **Our code** displays that QR code image URL

**Example QR Code URL from NDI:**

```
https://staging.bhutanndi.com/qr/abc-123-def-456
```

This URL points to an image hosted by NDI, not by us.
